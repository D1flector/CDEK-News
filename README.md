# CDEK News - Тестовое задание

Динамический виджет новостей, разработанный в рамках тестового задания для СДЭК. Приложение построено на современном стеке с акцентом на чистую архитектуру, производительность и удобство пользователя.

## Ссылка на демо

https://cdek-news.vercel.app/

## Технологический стек

- **Фреймворк:** Next.js 15
- **Язык программирования:** TypeScript
- **Стили:** Tailwind CSS
- **Иконки:** Tabler Icons

## Инструкция по запуску

1. Клонировать репозиторий:

   ```bash
   git clone https://github.com/D1flector/CDEK-News.git
   ```

2. Установить зависимости:

   ```bash
   npm install
   ```

3. Запустить сервер разработки:

   ```bash
   npm run dev
   ```

4. Открыть `http://localhost:3000` в браузере.

## Принятые решения и архитектура

- **Обход Mixed Content (`route.ts`):** Прямые запросы из браузера к HTTP-серверу API вызывают ошибку безопасности. Приложение выступает посредником: оно само обращается к внешнему API сервера СДЭК и безопасно отдает данные на клиент.
- **Изоляция логики (`useNews.ts`):** Вся работа с API, пагинацией и состояниями (загрузка/ошибка) вынесена в кастомный хук. Там же происходит нормализация данных: хук сам склеивает неполные пути картинок от API в рабочие URL.
- **Разделение компонентов:**
  - `NewsWidget` (контейнер): вызывает хук, рендерит состояния (Skeleton-загрузка, ошибка, пустой список) и управляет пагинацией.
  - `NewsCard` (UI): универсальный компонент. За счет пропсов (`variant`, `isFirst`) он перестраивается под нужный макет (с большой картинкой, маленькой сбоку или без нее).
- **Оптимизация изображений:** Использован компонент `next/image`. Для первой картинки рубрики задан `priority` для мгновенной загрузки, остальные подгружаются лениво (`lazy`) при скролле.
- **Адаптивность:** Вёрстка реализована на Tailwind CSS по принципу Mobile-first. Сетка карточек перестраивается в зависимости от ширины экрана (колонкой на мобилках, с картинкой сбоку на десктопе).

## Реализованные дополнительные требования

- **Типизация:** Весь проект написан на TypeScript. Описаны строгие интерфейсы для ответов API (`NewsResponse`), объектов новостей (`NewsItem`) и пропсов всех компонентов.
- **Ленивая загрузка (Lazy Loading):** Разделил приоритеты загрузки. Первая картинка в списке загружается мгновенно (флаг `priority`), а все остальные обложки подгружаются лениво (`loading="lazy"`) только в тот момент, когда пользователь доскроллит до них.
